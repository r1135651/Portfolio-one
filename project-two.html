Redbird Flight Simulator

Calibration Software Redesign

Problem

Flight Simulator Software Redesign

Redbird flight simulator was spending a significant amount of money paying an engineer to troubleshoot user problems when users were annually calibrating their flight simulators. The user interface for the calibration was not offering support or clarity to the user so they could not resolve their own problems. Redbird wanted a user interface solution to empower users to resolve their own problems.

Roles: UX Researcher, Visual Designer

Goals

  • Reduce the time/cost obligation of Redbird in resolving user problems
  • Update Design Language

User Research

Initially I began to define what the contributing factors were to Redbirdâ€™s user calibration problems.

Combining User data, stakeholder requests and heuristic analysis I created a priority list of requirements for my redesign:

  1. Reduce the time/cost obligation of Redbird in resolving user problems
  2. Providing information for users to identify and solve their own problems
  3. Improve test display, identifying recent tests, test status
  4. Give clear status of system and peripherals with detailed instructions for installing and using them
  5. Provide information about completion and due dates
  6. Ease repetitive user tasks

Understanding the User

I created a person to created a shared understanding of who our user was between stakeholder, engineer, developer and designer.

Ideation

I wanted to create a central screen or dashboard to summarize the calibration status and give a clear navigation method.

  • Tabs Menu - Ended up being too confusing for users to identify where flight tests were displayed and cumbersome to move between screens.
  • Cards UI - Just defer the problem of how to display the many calibration tests without giving the user much useful dashboard information.
  • List Menu UI - Held the most promise in providing useful dashboard information and providing a means of navigating through the plethora of calibration tests.
